只读验收建议：

2. 检查 `/openapi.json` 的路由是否存在。
3. 使用已有项目 `ref-001` 从 8788 下载交付包。
4. 不要用 `海外视频本地化MVP/scripts/walkthrough_check.py` 做日常只读验收；该脚本会真实生成脚本并写入成稿库、反馈库。

## 后续模型介入建议

1. 先把 8788 的 FastAPI 作为统一业务入口。
2. 新增模型供应商时，在脚本生成层做 provider 适配，不要让前端直接持有 API Key。
3. 将“参考素材、产品资料、模板、历史反馈”组合成结构化上下文，再调用模型。
4. 把模型输入、模型名、提示词版本和输出校验结果写入项目快照，便于复现。
5. 在切换 MySQL 为主数据源前，先完成一次文件与数据库的差异清单和单向迁移方案。
