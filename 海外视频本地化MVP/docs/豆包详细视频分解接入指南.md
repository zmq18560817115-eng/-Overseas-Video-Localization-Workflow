# 豆包详细视频分解接入指南（给 Cursor / Codex）

本文档用于把当前“前期素材解析 / 结构拆解”从规则模板升级为火山方舟豆包模型驱动的精细视频拆解。目标效果参考“小红书视频拆解器”：完整逐字稿、摘要、分镜时间轴、画面描述、台词、字幕/标题、结构角色都能落到 JSON 和页面。

## 一句话结论

不要用 SeedDance 做前期解析。SeedDance 是视频生成模型，继续留在后面的 AI 空镜/分镜视频生成环节。

前期素材解析应接：

- 视频理解主模型：`doubao-seed-2-1-turbo-260628`
- 高精度备用模型：`doubao-seed-2-1-pro-260628`
- 语音转写：火山语音 ASR / 语音转字幕接口

推荐默认链路：

```text
TikTok 视频文件或可访问 URL
  -> 抽音频 / 上传视频
  -> ASR 生成逐字稿与时间戳
  -> 豆包视频理解生成分镜、画面描述、结构标签
  -> 规则合并与校验
  -> 写入 AI拆解结果/{link_id}/analysis.json、shots.json、transcript.json、analysis.md
  -> 页面展示完整文案与分镜表
```

## 当前项目现状

当前 `decompose` 只是规则模板：

- 主要从标题、话题、元数据推断。
- 输出 8 字段：`hook_3s`、`pain_points`、`selling_points`、`scenes`、`video_structure`、`subtitle_layout`、`cta`、`reusable_template`。
- 产物路径：`海外视频本地化MVP/AI拆解结果/{link_id}/analysis.json`
- 当前 `analyze_provider` 多数为 `rule`。

升级时必须保留这 8 字段，避免影响脚本生成页；新增精细字段单独存储。

## 模型如何分工

### 1. ASR：负责“听懂人说了什么”

ASR 输出：

- `full_transcript`：完整逐字稿。
- `segments[]`：每段台词的 `start`、`end`、`text`。
- 可选：识别语种、置信度、说话人。

为什么不用视频理解模型单独做逐字稿：视频理解会总结得更顺，但逐字稿和时间戳容易漂。ASR 单独跑，后面再交给豆包视频理解合并，稳定很多。

### 2. 豆包视频理解：负责“看懂画面与结构”

默认模型：

```text
doubao-seed-2-1-turbo-260628
```

使用场景：

- 日常批量拆竞品素材。
- 20 秒到 3 分钟短视频。
- 需要平衡速度、成本和稳定性。

高精度模型：

```text
doubao-seed-2-1-pro-260628
```

使用场景：

- 爆款重点样本。
- 字幕密集、画面复杂、长视频。
- turbo 输出结构不稳定时重跑。

豆包输入：

- 视频 URL，或通过火山方舟 File API 上传后的文件引用。
- ASR 转写片段。
- 现有 TikTok 元数据：标题、作者、点赞、收藏、评论、话题标签。
- 固定输出 schema。

豆包输出：

- 分镜时间轴。
- 每镜画面描述。
- 每镜台词/字幕。
- 结构角色：`hook`、`pain`、`demo`、`proof`、`comparison`、`cta`、`transition`。
- 可复用模板与拍摄建议。

### 3. 规则合并层：负责“变成系统可用的数据”

模型不要直接改 CSV。Cursor 实现时应写一个合并函数：

1. 校验 JSON 是否完整。
2. 修正时间轴重叠或空洞。
3. 从精细分镜反推原来的 8 字段。
4. 写入文件。
5. 失败时回退到现有 `rule` 解析。

## 建议环境变量

不要把密钥写进源码。优先放在 `海外视频本地化MVP/.env` 或 Windows 用户环境变量。

```env
# 火山方舟 Ark，大模型视频理解使用
ARK_API_KEY=

# 默认详细视频分解模型
DOUBAO_VIDEO_ANALYSIS_MODEL=doubao-seed-2-1-turbo-260628

# 高精度备用模型
DOUBAO_VIDEO_ANALYSIS_MODEL_PRO=doubao-seed-2-1-pro-260628

# auto: 默认 turbo，失败或重点素材再用 pro；turbo/pro: 强制指定
DOUBAO_VIDEO_ANALYSIS_MODE=auto

# 是否启用 ASR；没配 ASR 时仍可做画面拆解，但逐字稿质量会下降
DOUBAO_ASR_ENABLED=1

# ASR 凭证按火山控制台实际字段填写，禁止提交到 Git
VOLCENGINE_ASR_APP_ID=
VOLCENGINE_ASR_ACCESS_TOKEN=
VOLCENGINE_ASR_CLUSTER=
```

如果已经有 `ARK_API_KEY` 给 SeedDance Ark 使用，可以复用同一个 key，但要确认账号已开通对应模型与配额。

## 目标输出文件

每个素材建议写入这些文件：

```text
海外视频本地化MVP/AI拆解结果/{link_id}/
  analysis.json       # 兼容旧 8 字段 + 新增摘要
  transcript.json     # ASR 原始逐字稿
  shots.json          # 精细分镜表
  analysis.md         # 给人看的拆解报告
  raw_response.json   # 可选，调试用；不要包含密钥
```

`analysis.json` 继续保留旧字段，新增：

```json
{
  "analyze_provider": "doubao_video",
  "analyze_model": "doubao-seed-2-1-turbo-260628",

  "full_transcript": "完整逐字稿，可从 transcript.json 同步",
```

## Cursor 施工步骤

### 第 1 步：新增两个模块

建议新增：

```text
海外视频本地化MVP/app/doubao_video_analysis.py
海外视频本地化MVP/app/volcengine_asr.py
```

`volcengine_asr.py` 只负责：

- 输入本地视频或音频路径。
- 返回 `segments[]` 和 `full_transcript`。
- 失败时返回清晰错误，不中断规则模板兜底。

`doubao_video_analysis.py` 只负责：

- 读取 `.env`。
- 调用火山方舟 Chat / 视频理解。
- 强制模型输出 JSON。
- 校验并保存精细结果。

### 第 2 步：改 `scripts/pipeline.py`

给 `decompose` 增加 provider：

```text
python scripts/pipeline.py decompose --provider rule
python scripts/pipeline.py decompose --provider doubao --limit 1
python scripts/pipeline.py decompose --provider doubao --model pro --link-id 12
```

实现要求：

- 默认仍可用 `rule`，避免没 key 时整个流程坏掉。
- `--provider doubao` 时优先跑 ASR，再跑视频理解。
- 没有本地视频文件时，如果 TikTok URL 可访问，用 URL；不可访问则提示先下载素材。
- 成功后 `analyze_provider=doubao_video`。
- 失败后写 `error_message`，并可回退规则模板。

### 第 3 步：改页面读取

当前素材页只读 8 字段。后续可让前端额外读取：

```text
AI拆解结果/{link_id}/shots.json
AI拆解结果/{link_id}/transcript.json
```

页面展示目标：

- 上方：完整逐字稿和摘要。
- 下方：分镜表，列为 `# / 时间 / 画面描述 / 台词 / 字幕或标题 / 结构角色`。
- 右侧保留原 8 字段，给脚本生成继续用。

### 第 4 步：提示词固定

给豆包的视频理解提示词应固定，不要每次临时拼。

推荐系统提示词：

```text
你是短视频素材拆解分析师。你只基于提供的视频、ASR逐字稿和元数据分析，不得编造画面中不存在的信息。输出必须是合法 JSON，不要 markdown。时间码使用 mm:ss。若无法确认某项，填空字符串或低 confidence。
```

推荐用户提示词核心：

```text
请把这个短视频拆成逐镜脚本。要求：
1. 按镜头或语义段切分，通常 2-8 秒一段。
2. 每段必须包含 start、end、visual_description、dialogue、subtitle_or_title、scene_type、camera_action、structure_role、reuse_note、confidence。
3. structure_role 只能从 hook、pain、demo、proof、comparison、cta、transition、other 中选择。
4. dialogue 优先使用 ASR 原文；不要润色成新文案。
5. visual_description 只描述画面，不写营销推断。
6. 最后给 summary、reusable_template、subtitle_layout、video_structure。
7. 只输出 JSON。

ASR:
{transcript_segments}

元数据:
{metadata}
```

### 第 5 步：验收标准

Cursor 写完后必须通过：

```powershell
cd C:\Users\bu\Documents\海外视频本地化工作流\海外视频本地化MVP
.venv\Scripts\python.exe scripts\pipeline.py decompose --provider doubao --limit 1
```

人工检查：

- 至少 5 个分镜段。
- 每段有时间、画面描述、台词或字幕。
- `analysis.json` 仍有旧 8 字段。
- 页面上旧脚本生成不报错。

健康检查：

```powershell
Invoke-RestMethod http://127.0.0.1:8788/api/health
```

`decompose.label` 后续应显示类似：

```text
结构拆解（豆包视频理解 + ASR，规则兜底）
```

## 和现有脚本生成的关系

精细分镜不要直接替代脚本生成。正确关系是：

```text
详细视频分解 = 学竞品结构、节奏、镜头、字幕密度
脚本生成 = 结合我方产品标签，生成我方新视频脚本
SeedDance = 根据我方脚本生成 AI 空镜或分镜视频
```

也就是说，豆包详细分解只作为“参考视频拆解层”，不能把竞品文案原样复制到我方脚本里。

## 不要做的事

- 不要把 `ARK_API_KEY`、ASR token、任何密钥写进源码或提交。
- 不要删除现有 `rule` 结构拆解。
- 不要让前端直接持有 API Key。
- 不要让模型直接写 CSV；先落 JSON，再由代码同步索引。
- 不要把竞品台词当作我方脚本文案直接复用。
