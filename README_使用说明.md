# 海外视频本地化工作流

本目录集中管理 **竞品对标 → 脚本出稿 → 交付剪辑** 的完整链路。

## 一键启动（推荐）

双击 **`启动工作台.cmd`** → 打开 **http://127.0.0.1:8788**

### 工作台五个页面

| 页面 | 作用 |
|------|------|
| **素材库** | 浏览 TikTok 竞品视频、查看 8 字段结构拆解、播放量 |
| **脚本生成** | 选产品与结构参考 → 四类标签 → 生成脚本 → 完成交付 → 下载 zip |
| **成稿库** | 已交付项目列表与 zip 下载 |
| **反馈库** | 剪辑/运营反馈与投放数据 |
| **设置** | 同步 TikTok、结构拆解、模板、产品资料；SeedDance 配置 |

产品资料长文案编辑：设置 → **打开产品资料维护**（非日常必经步骤）。

## 引擎说明（谁在干活）

| 环节 | 依赖 |
|------|------|
| 同步 TikTok | TikTok oEmbed + Playwright（抓元数据，不看视频内容） |
| 结构拆解 | **规则模板**（标题/话题 → 8 字段，非大模型） |
| 生成脚本 | **Claude**（配置 `ANTHROPIC_API_KEY`）或 **规则模板**兜底 |
| 完成交付 | `overseas-loc-mvp` 子进程（字幕、zip、合规校验） |
| AI 空镜 | **SeedDance 2.0** via fal.ai（配置 `FAL_KEY`，可选） |

## 交付 zip 内容（给剪辑 / 拍摄）

| 文件 | 说明 |
|------|------|
| `交付脚本包.md` | 七项脚本（标题 / 口播 / 分镜 / SeedDance 等） |
| `交付脚本包.json` | 同上，机器可读 |
| `subtitles.srt` | 英文字幕，PR / 剪映直接导入 |
| `剪辑单.html` | 5 镜剪辑对照（浏览器打开） |
| `broll/shot-*.mp4` | SeedDance 空镜（配置 `FAL_KEY` 后自动生成） |

## 应用目录

| 目录 | 用途 |
|------|------|
| `海外视频本地化MVP/` | **8788 主工作台**（唯一日常入口） |
| `overseas-loc-mvp/` | **内部交付引擎**（API + 子进程，不单独开页面） |
| `成稿库/` | 已交付项目索引（CSV + JSON） |
| `反馈库/` | 剪辑/运营反馈 |

每条出稿项目保存在：`overseas-loc-mvp/runs/{slug}/`

## 环境配置

1. 首次使用：双击 **`安装并检查开发环境.cmd`**
2. **脚本生成（可选）**：Windows 用户环境变量或 `.env` 配置 `ANTHROPIC_API_KEY`
3. **SeedDance（可选）**：`overseas-loc-mvp/.env` 填写 `FAL_KEY`，或运行 **`配置SeedDance.cmd`**
4. 知识库：根目录 **`config/knowledge-sources.json`**

未填 `FAL_KEY` 时：脚本与交付包可完整跑通；仅 AI 空镜 mp4 需另行配置。

## 手工归档文件夹（可选）

| 文件夹 | 用途 |
| --- | --- |
| `00_inbox` ~ `09_archive` | 视频制作各阶段归档 |
| `assets` | 字体、术语表 |
| `templates` | 归档映射模板 |

验收通过后：`归档MVP产出.cmd night-pumping-v1`（映射见 `templates/MVP产出归档映射.md`）。
